# Schema Audit: is-0.9.json vs ToIntermediateSchema Implementations

> 2026-03-10, generated by [claude-opus-4.6](https://www-cdn.anthropic.com/0dd865075ad3132672ee0ab40b05a53f14cf5288.pdf)

Issue: span-79q

## 1. Schema vs Struct Mismatches

### 1.1 Type Mismatch: `finc.mega_collection`

The JSON schema defines `finc.mega_collection` as `"type": "string"`, but the Go struct
uses `[]string`. Every implementation sets this as a slice (e.g., `[]string{"JSTOR"}`).
Either the schema should be `"type": "array"` or the struct should be `string`.

- **Schema**: `schema/is-0.9.json:8216` — `"type": "string"`
- **Struct**: `formats/finc/intermediate.go:104` — `MegaCollections []string`

### 1.2 Fields in Struct but NOT in Schema

The schema has `"additionalProperties": false`, meaning any field not listed in
`properties` will cause validation failure. These struct fields are absent from the schema:

| JSON Tag | Go Field | Go Type |
|---|---|---|
| `finc.id` | ID | string |
| `rft.part` | Part | string |
| `x.date` | Date | time.Time |
| `x.indicator` | Indicator | string |
| `x.packages` | Packages | []string |
| `x.labels` | Labels | []string |
| `x.oa` | OpenAccess | bool |
| `x.license` | License | []string |
| `x.footnotes` | Footnotes | []string |

Every record will fail schema validation if any of these are emitted (and `omitempty`
won't help for non-zero defaults like `x.date`'s zero time).

## 3. Constraint Violations

### 3.1 Non-numeric Page Fields

Schema requires `rft.spage`, `rft.epage`, `rft.tpages` to match `^[0-9]*$`.

| Source | Field(s) | Issue |
|---|---|---|
| **jats** (and jstor, degruyter, ios) | spage, epage | Raw XML `<fpage>`/`<lpage>` — may contain "e45", "S12", "ix" |
| **thieme** | spage, epage | Raw XML, no validation |
| **ieee** | spage, epage | Raw XML attributes, no validation |
| **ceeol/article** | spage, epage, tpages | Raw XML, no validation |
| **doaj/next** | spage, epage | Raw JSON, no validation |
| **doaj/document** | spage, epage | Raw JSON, no validation |
| **crossref** | spage, epage, tpages | Defaults to `"0"` when unparseable (numeric but semantically wrong) |

Sources with safe page handling: doaj/oaidc (regex extraction), ssoar (regex), genderopen (regex).

Sources: jats (and embedders), ieee. The `Pages` field becomes `"-"` via
`fmt.Sprintf("%s-%s", "", "")` when both start/end pages are empty.

## 4. Other Bugs

### 4.1 crossref: `finc.record_id` Never Set

`formats/crossref/document.go` — `output.RecordID` is never assigned. This is a required
field. The `output.ID` field is set but `RecordID` is not.

### 4.2 ieee: Genre Constant Defined but Not Used

`formats/ieee/publication.go:24` defines `Genre = "article"` but the ToIntermediateSchema
method never assigns it to `output.Genre`.

### 4.3 ieee: Date Parsing Bug

`formats/ieee/publication.go:231-234` — When parsing `date.Day`, the result is mistakenly
assigned to `m` (month) instead of `d` (day), corrupting dates.

### 4.4 ssoar: `FindPages` Bug

`formats/ssoar/record.go:169-178` — In the default branch, `StartPage` is set to
`sms[0][0]` (full regex match like `"87-101"`) instead of the first capture group. This
produces non-numeric start pages.

### 4.5 ssoar: `log.Fatal` on Date Parse Failure

`formats/ssoar/record.go:228` — Calls `log.Fatal(err)` which terminates the entire
process on a single bad date instead of skipping the record.

### 4.6 doaj/document: RecordID = DOI (May Be Empty)

`formats/doaj/document.go` — `RecordID` is set to `doc.DOI()`. If the document has no
DOI, the required field will be empty. Should fall back to `doc.ID`.

### 4.7 mediarep: Both ID and RecordID Never Set

`formats/mediarep/dim.go` — Neither `output.ID` nor `output.RecordID` is set.

### 4.8 highwire: `finc.mega_collection` Conditional

`formats/highwire/record.go` — `MegaCollections` is only set when `dc:publisher` is
non-empty. If missing, the required field will be absent.

### 4.9 zvdd: Publisher Overwrite Loop

`formats/zvdd/mets.go` — `output.Publishers` is overwritten in a loop; only the last
origin's publishers survive.

### 4.10 genderopen: Empty Language in Slice

`formats/genderopen/record.go` — Sets `Languages` to `[]string{record.Metadata.Dc.Language}`.
If the language field is empty, this produces `[""]` instead of `nil`/`[]`.

## 5. Summary

**Critical issues** (data loss or required field violations):
- Schema/struct type mismatch on `finc.mega_collection` (string vs []string)
- 9 struct fields absent from schema (with `additionalProperties: false`)
- 6 sources missing required fields (imslp, dblp, mediarep, crossref, ieee, ssoar)
- ieee date parsing bug corrupts dates
- ssoar `log.Fatal` kills process on bad data

**Moderate issues** (constraint violations):
- 7+ sources pass through non-numeric page values
- Several sources hardcode wrong genre/type for their content
- crossref page fields default to "0" instead of empty

**Minor issues** (cosmetic/semantic):
- Pages field becomes "-" when empty
- hhbd genres all map to "unknown"
- zvdd DublinCore struct has typo "DublicCoreRecord"
